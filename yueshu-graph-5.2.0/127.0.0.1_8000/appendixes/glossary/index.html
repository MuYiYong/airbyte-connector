    术语表¶

    本术语表列举了悦数图数据库和 GQL 中特定的术语及其含义。

    agent¶

    一个在悦数图数据库集群的每台服务器上运行的守护进程，用于简化分布式服务组的管理。

    Catalog¶

    GQL 中用于存储 Schema 的系统对象。每个集群有一个 Catalog。

    Compact¶

    一个通过减少存储碎片化、合并小文件和删除过时数据来优化存储空间的过程。

    Flush¶

    一个将 RocksDB 内存中的数据写入磁盘的过程。

    Follower¶

    在 Raft 组中非 Leader 角色的副本，起到备份的作用。更多信息，请参见数据平衡。

    GQL¶

    GQL，即图查询语言（Graph Query Language），是用于与悦数图数据库交互的查询语言。

    graphd¶

    graphd，也称为nebula-graphd，是 Graph 服务的进程名。

    Graph 服务¶

    作为查询引擎，处理图查询请求的悦数图数据库服务。

    gRPC¶

    gRPC，即 Google 远程过程调用（Google Remote Procedure Call），是一种高性能、开源、通用的 RPC 框架，用于连通集群中的悦数图数据库服务。

    LineString¶

    一种地理空间类型，表示由多条直线连接的点在球面上构成的路径。

    Leader¶

    在 Raft 组中负责处理读写请求的副本。每个 Raft 组只有一个 Leader 副本。

    metad¶

    metad，也称为nebula-metad，是 Meta 服务的进程名。

    Meta 服务¶

    作为核心控制组件，负责管理图元数据、数据分布和集群配置的悦数图数据库服务。

    ngctl¶

    一个用于管理悦数图数据库的集群、服务组、主机、服务和用户的命令行工具。

    ngql¶

    一个用于与 Graph 服务交互并执行 GQL 查询的命令行工具。

    Peer¶

    分片的副本，可以是 Leader 副本或 Follower 副本。更多信息，请参见 SHOW PEERS。

    Point¶

    一种地理空间类型，表示球面上的单个点，由经度和纬度坐标定义。

    Polygon¶

    一种地理空间类型，表示一个具有闭合边界的形状，该形状内可以包含 0 个或多个孔。它由一系列构成外边界的点，以及构成这些孔的点共同定义。

    Raft¶

    一种共识算法，用于在悦数图数据库中确保每个服务组内的数据一致性和高可用性。更多信息，请参见 Raft。

    RocksDB¶

    一个高可用的嵌入式键值存储系统，用作悦数图数据库的存储引擎。更多信息，请参见 RocksDB。

    Schema¶

    用于存储图类型、图和过程等资源对象的逻辑容器。更多信息，请参见 CREATE SCHEMA。

    storaged¶

    storaged，也称为nebula-storaged，是 Storage 服务的进程名。

    Storage 服务¶

    悦数图数据库的数据存储层，负责持久化存储图数据，包括点和边。更多信息，请参见 Storage 服务。

    TLS¶

    传输层安全协议（Transport Layer Security 的简称）是一种用于在网络上提供安全通信的协议。更多信息，请参见为悦数图数据库服务启用 TLS。

    边¶

    边，也称为关系，表示图中两个点之间的连接。

    边类型¶

    相同类型的边具有相同的属性类型、标签以及约束，比如多边键。

    边模式¶

    边模式用于根据特定条件匹配图中的边，比如起始点和目标点、边类型、边的方向、属性和标签等。

    变量¶

    用于在 GQL 查询中存储数据值以供数据检索和数据操作的标识符。更多信息，请参见变量概述。

    表达式¶

    一个由常量、变量、运算符、函数调用等组成的声明，其的计算结果为单个值。表达式是一个查询的基本构建单元。

    标签¶

    与一个或多个点类型或边类型相关联的标记。每个点类型或边类型可以有一个或多个标签。

    标识符¶

    一个用于标识对象（如图类型、点类型或点属性）的名称。

    参数¶

    由名称和值成对组成，用于在查询中传递值。更多信息，请参见配置参数。

    常量¶

    一个保存固定值的变量。

    查询¶

    用于检索或者操作存储在数据库中的数据的请求。查询可包含插入、更新或者删除数据等操作。

    插件¶

    运行时可插拔的组件，为开发者提供了可编程接口。更多信息，请参见插件概述。

    点¶

    点，也称为顶点，表示图中的实体。

    点类型¶

    相同类型的点具有相同的属性类型、标签以及约束，比如主键。

    点模式¶

    点模式用于根据特定条件匹配图中的点，比如点类型、标签和属性等。

    多边键¶

    一个标识两个点之间相同类型的边的唯一标识符。该标识符是在两点之间创建平行边的先决条件。更多信息，请参见参数multiedge_key_list。

    分片¶

    在悦数图数据库服务组中，分片是数据分布的基本单位。每个分片包含一个或多个副本。

    副本¶

    副本（也称为分片副本）是分片的备份。一个分片的所有副本构成一个 Raft 组。更多信息，请参见数据平衡。

    服务组¶

    一个服务组由一个或多个 Graph 服务和一个或多个 Storage 服务组成，用于处理特定业务的数据和查询。更多信息，请参见服务组。

    关键字¶

    一个在 GQL 中具有特殊含义的字。

    过程¶

    预定义且可重用的一组指令，用于根据入参执行特定的计算任务，生成结果并可选地修改数据对象。更多信息，请参见过程概述。

    函数¶

    一个基于预定义的算法，将一组入参映射为返回值的命令。

    函数签名¶

    函数签名定义了函数的名称、返回类型和入参类型。

    会话¶

    来自单个客户端的一系列连续的 GQL 请求。

    会话参数¶

    在会话上下文中设置的参数，可在当前会话中的所有请求中使用。更多信息，请参见使用会话参数。

    集群¶

    一组运行悦数图数据库服务的服务器，共同提供分布式图数据库服务。

    每个集群至少包含一个 Meta 服务和一个服务组。

    连接器¶

    一种用于在悦数图数据库和大数据处理框架（如 Apache Spark 和 Apache Kafka）间实现数据传输的组件。

    路径¶

    路径由一个非空的点序列和一个（可能为空的）边序列组成。当存在两个或更多的点时，路径是从一个点开始，并由一系列有序的（边、点）对构成的图元素序列。

    路径类型¶

    路径类型指定了要匹配的路径的类型。路径的类型包括WALK、TRAIL、ACYCLIC和SIMPLE。更多信息，请参见路径类型前缀。

    路径类型前缀¶

    路径类型前缀由路径类型和一个可选的后缀（PATH或PATHS）组成。

    路径模式¶

    由语法元素组成的表达式，用于匹配路径。更多信息，请参见路径模式。

    路径搜索前缀¶

    路径搜索前缀指定了要返回的路径数量，可能包含以下关键字之一：ALL、ANY、ALL SHORTEST、ANY SHORTEST和SHORTEST。此外，还可以指定一个可选的路径数量和一个可选的路径类型。更多信息，请参见路径搜索前缀。

    命名过程¶

    一个在 GQL Catalog 中定义、可以在不同查询中通过名称调用的过程。

    模块¶

    一个包含一组用于定义命名过程的代码的文件。

    模式¶

    模式（也称为图模式）是由一个或多个路径模式组成的集合。

    内联过程¶

    一个在查询中定义且仅在该查询中有效的过程。

    请求参数¶

    在请求上下文中设置的参数，仅对当前请求有效。更多信息，请参见设置请求参数。

    入参¶

    在函数或过程被调用时传递给它的值。

    数据模型¶

    数据模型定义了数据库中数据的结构。在悦数图数据库中，数据模型具有属性图结构。

    属性¶

    由名称和值成对组成，用于在图中存储数据。

    属性名称在同一点类型或边类型内是唯一的，而属性值必须是支持的数据类型的值。更多信息，请参见数据类型。

    索引¶

    一种通过快速访问特定数据从而提高数据库读取性能的数据结构。

    索引提示¶

    一条查询中用于指导查询优化器使用或不使用哪个索引的注释。

    图¶

    图（属性图的简称），是由点和边组成的数据结构，其中一些点由边连接。点和边都可以具备属性和标签。

    图分析 Operator¶

    图分析 Operator（悦数图分析 Operator 的简称）是一个用于部署和管理图分析作业的编排工具。更多信息，请参见什么是悦数图分析 Operator。

    图管理¶

    图管理（悦数图管理的简称）是一个基于浏览器的图数据库可视化工具，用于管理和查询由悦数图数据库托管的数据。更多信息，请参见什么是悦数图管理。

    图类型¶

    相同类型的图具有相同的图元素类型。

    图模式¶

    一组路径模式。

    图数据库 Operator¶

    图数据库 Operator（悦数图数据库 Operator 的简称）是一款编排工具，用于自动化部署和管理悦数图数据库服务组。该工具基于 Kubernetes Operator 模式构建。更多信息，请参见什么是悦数图数据库 Operator。

    图探索¶

    图探索（悦数图探索的简称）是一款 Web 工具，提过可视化的界面帮助用户探索关系并挖掘托管在悦数图数据库中的数据的价值。更多信息，请参见什么是悦数图探索。

    临时图¶

    临时图基于现有的图或子图创建，存在于宿主服务的内存中。

    图元素¶

    一个点或边。

    谓词¶

    用于指定一个条件的布尔表达式，该条件可能为真、假或未知。更多信息，请参见谓词。

    悬挂边¶

    两端或一端未连接点的边。

    向量索引¶

    一种用于加速向量之间相似性搜索查询的索引。更多信息，请参见 CREATE VECTOR INDEX。

    语法标记¶

    用于定义 GQL 语言结构和语法的一组规则。更多信息，请参见语法表示法。

    语句¶

    一个可以独立执行的查询单元。

    语句块¶

    一组语句的集合，这些语句可以执行数据修改操作或数据检索操作。更多信息，请参见语句块。

    运维监控¶

    运维监控（悦数运维监控的简称）是一个基于浏览器的图数据库可视化工具，用于管理和监控悦数图数据库集群。更多信息，请参见什么是悦数运维监控。

    悦数图数据库¶

    一个适合存储和查询大规模图结构数据的图数据库，具备分布式、可扩展和高可用等特性。

    执行计划¶

    执行计划描绘了执行一个查询所需的一系列操作。

    主机¶

    一个主机上运行一个或多个悦数图数据库服务。

    主键¶

    用于唯一标识点的属性或属性集合。更多信息，请参见参数PRIMARY KEY。

    主 Schema¶

    与用户关联的默认 Schema。

    主图¶

    与用户关联的默认图。

    子图¶

    图的子集，由一组点和边组成。

    字面量¶

    在查询中直接写入的固定值。

    作业¶

    一组由 Meta 服务分发给 Storage 服务的任务。作业类型包括 Compact、Flush 等。
