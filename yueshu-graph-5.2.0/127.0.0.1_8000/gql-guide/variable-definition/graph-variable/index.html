    图变量¶

    图变量用于存储使用 MATCH 语句检索得到的子图。关键字GRAPH用于声明图变量。

    在定义图变量时，可以预定义子图所包含的点类型和边类型，再使用 MATCH 语句检索预定义类型的子图。

    或者，也可以在定义图变量时不指定点类型和边类型，直接使用 MATCH 语句检索子图。

    前提条件¶

    • 已创建图。更多信息，请参见 CREATE GRAPH。
    • 具备在当前 Schema 执行MATCH ANY GRAPH操作的权限，或具备在所查询的图上执行MATCH操作的权限。更多信息，请参见角色与权限。

    语法¶

    graph_variable_definition

    graph_variable_definition ::=
     'PROPERTY'? 'GRAPH' graph_variable 
        ( 'TYPED'? 'GRAPH' '{'
        (
            (('NODE' | 'VERTEX') 'TYPE'? node_type_name '(' node_type_filler ')')
            | 
            (('EDGE' | 'RELATIONSHIP') 'TYPE'? edge_type_name  
                (
                    '(' source_node_type_name ')' '-[' edge_type_filler ']->' '(' destination_node_type_name ')'
                    | 
                    '(' destination_node_type_name ')' '<-[' edge_type_filler ']-' '(' source_node_type_name ')'
                    | 
                    '(' source_node_type_name ')' '~[' edge_type_filler ']~' '(' destination_node_type_name ')'
                )
            )
        )
        (
            ',' 
            (
                (('NODE' | 'VERTEX') 'TYPE'? node_type_name '(' node_type_filler ')')
                | 
                (('EDGE' | 'RELATIONSHIP') 'TYPE'? edge_type_name  
                    (
                        '(' source_node_type_name ')' '-[' edge_type_filler ']->' '(' destination_node_type_name ')'
                        | 
                        '(' destination_node_type_name ')' '<-[' edge_type_filler ']-' '(' source_node_type_name ')'
                        | 
                        '(' source_node_type_name ')' '~[' edge_type_filler ']~' '(' destination_node_type_name ')'
                    )
                )
            )
        )*                     
    '}' )? '=' 'GRAPH' '{' use_clause? match_statement return_statement '}'

    参数¶

    graph_variable¶

    要定义的图变量的名称。graph_variable可以是常规标识符或使用反引号(`)包围的分隔标识符。有关标识符的更多信息，请参见标识符命名规则。

    node_type_name¶

    待定义的图变量中的点类型的名称。此处指定的点类型必须在后续的 MATCH 及 RETURN 语句中指定。

    node_type_filler¶

    待定义的图变量中的点类型的属性与标签。

    此处指定的属性与标签必须与后续 MATCH 及 RETURN 语句中指定的相应点类型的属性与标签保持一致。

    关于此参数的更多信息，请参见 CREATE GRAPH TYPE 中的语法图。

    edge_type_name¶

    待定义的图变量中的边类型的名称。此处指定的边类型必须在后续的 MATCH 及 RETURN 语句中指定。

    source_node_type_name¶

    待定义的图变量中边类型的起始点类型的名称。此处指定的点类型必须在后续的 MATCH 及 RETURN 语句中指定。

    destination_node_type_name¶

    待定义的图变量中边类型的目标点类型的名称。此处指定的点类型必须在后续的 MATCH 及 RETURN 语句中指定。

    edge_type_filler¶

    待定义的图变量中边类型的属性与标签。

    此处指定的属性与标签必须与后续 MATCH 及 RETURN 语句中指定的相应边类型的属性与标签保持一致。

    关于此参数的更多信息，请参见 CREATE GRAPH TYPE 中的语法图。

    use_clause¶

    USE 子句指定在 MATCH 语句中使用的图。若要使用的图已设置为主图，则可省略该子句。更多信息，请参见 USE。

    match_statement¶

    MATCH 语句检索将存储在图变量中的子图，该子图必须至少包含一个点。

    如果指定了node_type_name或edge_type_name，则相应的点类型或边类型必须在 MATCH 语句中指定。

    更多信息，请参见 MATCH。

    return_statement¶

    RETURN 语句指定将存储在图变量中的子图，该子图必须至少包含一个点。

    如果指定了node_type_name或edge_type_name，则相应的点类型或边类型必须在 RETURN 语句中指定。

    更多信息，请参见 RETURN。

    示例¶

    以下示例基于示例数据集。由于图变量无法通过 RETURN 语句直接返回，因此示例使用过程algo.graph_stats展示存储在图变量中的子图的统计信息。

    在调用过程algo.graph_stats之前，需执行语句CREATE PLUGIN algo将内置插件algo加载到当前 Catalog 中。更多信息，请参见 CREATE PLUGIN。

    定义图变量¶

    '''
GRAPH g = GRAPH { USE movie MATCH (n) RETURN n } 
CALL algo.graph_stats(g) 
RETURN *
'''

    上述语句定义了一个图变量g，用于存储在 MATCH 语句中检索得到的所有点。

    返回：

    +-------------------+-------------------+
| total_edge_number | total_node_number |
+-------------------+-------------------+
| 0                 | 4589              |
+-------------------+-------------------+

    定义图变量时预定义点类型和边类型¶

    '''
GRAPH g TYPED GRAPH { 
    NODE TYPE Actor (LABEL Person {id INT64 PRIMARY KEY, name STRING, birthDate DATE } ), 
    NODE TYPE Movie (LABEL Movie {id INT64 PRIMARY KEY, name STRING }), 
    EDGE TYPE Act (Actor)-[LABEL Act{}]->(Movie) 
    } = GRAPH { USE movie MATCH (n@Actor)-[r@Act]->(m@Movie) RETURN n, m, r }  
CALL algo.graph_stats(g) 
RETURN *    
'''

    上述语句定义了一个图变量g，其中预定义了点类型Actor和Movie以及边类型Act。图变量g存储了 MATCH 语句中检索得到的Actor和Movie类型的点以及Act类型的边。

    返回：

    +-------------------+-------------------+
| total_edge_number | total_node_number |
+-------------------+-------------------+
| 3732              | 3533              |
+-------------------+-------------------+
