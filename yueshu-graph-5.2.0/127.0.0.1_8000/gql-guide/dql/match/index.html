    MATCH¶

    MATCH语句通过模式匹配从图或临时图中检索数据。有关模式匹配的更多信息，请参见模式。

    前提条件¶

    • 已创建图或临时图。更多信息，请参见 CREATE GRAPH 和 CREATE TEMPORARY GRAPH。
    • 若要检索图中的数据，须具备在当前 Schema 执行MATCH ANY GRAPH操作的权限，或具备在当前图中执行MATCH操作的权限。更多信息，请参见角色与权限。
    • 若要检索临时图中的数据，必须是具备角色ADMIN的用户或临时图的所有者。

    语法¶

    match_statement
    graph_pattern
    match_mode
    path_pattern
    path_mode_prefix
    path_search_prefix
    path_term
    path_factor
    parenthesized_path_pattern
    element_pattern
    graph_pattern_yield_clause

    match_statement ::=
    ( 'OPTIONAL' )? 'MATCH' graph_pattern graph_pattern_yield_clause?

graph_pattern ::= match_mode? path_pattern ( ',' path_pattern )* ( 'WHERE' search_condition )?

match_mode ::=
      'REPEATABLE' ('ELEMENTS' | 'ELEMENT' 'BINDINGS'?)
    | 'DIFFERENT' ( ('EDGE' | 'RELATIONSHIP') 'BINDINGS'? | ('EDGES' | 'RELATIONSHIPS') )

path_pattern ::=
    ( path_variable '=' )? ( path_mode_prefix | path_search_prefix )? path_term

path_mode_prefix ::=
    ( 'WALK' | 'TRAIL' | 'ACYCLIC' | 'SIMPLE' ) ( 'PATH' | 'PATHS' )?

path_search_prefix ::=
      'ALL' ( 'WALK' | 'TRAIL' | 'ACYCLIC' | 'SIMPLE' )? ( 'PATH' | 'PATHS' )?
    | 'ANY' ( number_of_paths )? ( 'WALK' | 'TRAIL' | 'ACYCLIC' | 'SIMPLE')? ( 'PATH' | 'PATHS' )?
    | 'ALL SHORTEST' ( 'WALK' | 'TRAIL' | 'ACYCLIC' | 'SIMPLE' )? ( 'PATH' | 'PATHS' )?
    | 'ANY SHORTEST' ( 'WALK' | 'TRAIL' | 'ACYCLIC' | 'SIMPLE')? ( 'PATH' | 'PATHS' )?
    | 'SHORTEST' number_of_paths ( 'WALK' | 'TRAIL' | 'ACYCLIC' | 'SIMPLE' )? ( 'PATH' | 'PATHS' )?

path_term ::=
      path_factor path_factor*

path_factor ::=
      element_pattern index_hint?
    | edge_pattern index_hint? graph_pattern_quantifier
    | parenthesized_path_pattern graph_pattern_quantifier

parenthesized_path_pattern ::=
    '(' ( path_variable '=' )? path_mode_prefix? path_term ( 'WHERE' search_condition )? ')'

element_pattern ::=
      node_pattern
    | edge_pattern

graph_pattern_yield_clause ::=
    'YIELD' ( element_variable_reference | path_variable_reference) ( ',' (element_variable_reference | path_variable_reference) )*

    参数¶

    OPTIONAL¶

    当使用关键字OPTIONAL时，即使图模式没有找到匹配项，查询仍会返回null作为其结果，查询会继续返回其他子句中可能匹配到的数据。否则，匹配语句将不返回任何数据。

    match_mode¶

    匹配类型，用于限制路径模式匹配到的路径中是否可以出现重复边。支持以下匹配类型：

    • REPEATABLE：如果匹配类型以关键字REPEATABLE开头，则路径模式匹配到的路径中可以出现重复边和重复点。这是默认的匹配类型。
    • DIFFERENT：如果匹配类型以关键字DIFFERENT开头，则路径模式匹配到的路径中不可以出现重复边。

    path_variable¶

    表示路径的变量。更多信息，请参见变量表达式。

    WALK | TRAIL | SIMPLE | ACYCLIC¶

    在路径匹配期间过滤路径的路径类型。支持以下路径类型：

    • WALK：匹配任何类型的路径，无论是否有重复的边或点。这是默认的路径类型。
    • TRAIL：匹配不包含重复边的路径。
    • ACYCLIC：匹配不包含重复点的路径。
    • SIMPLE：匹配除起始点外，不包含重复点的路径。

    Note

    如果匹配类型以关键字DIFFERENT开头且路径类型设置为WALK或未设置，则实际生效的路径类型为TRAIL。

    PATH | PATHS¶

    每种路径类型都有一个可选的后缀：PATH或PATHS，用于声明这是路径。

    ALL | ANY | ALL SHORTEST | ANY SHORTEST¶

    在路径匹配期间可以选择性地过滤路径的数量。支持通过以下关键字选择路径的数量：

    • ALL：默认选项，选择所有匹配模式的路径。
    • ANY：选择任意一条或多个匹配模式的路径。您可以添加一个非负整数（number_of_paths）来指定返回的最大路径数。
    • ALL SHORTEST：选择所有匹配模式的最短路径。
    • ANY SHORTEST：选择任意一条匹配模式的最短路径。
    • SHORTEST：选择任意一条或者多个最短路径。您可以添加一个非负整数（number_of_paths）来指定返回的最大的最短路径数。

    number_of_paths¶

    要返回的最大路径数。默认值为1。必须是非负整数。

    element_pattern¶

    要匹配的元素模式，可以是点模式或边模式。

    node_pattern¶

    要匹配的点模式。更多信息，请参见点模式。

    edge_pattern¶

    要匹配的边模式。更多信息，请参见边模式。

    index_hint¶

    为查询优化器在执行查询时提供索引使用建议的索引提示。更多信息，请参见索引提示。

    graph_pattern_quantifier¶

    匹配边模式次数的上下限。

    支持以下格式：

    • {lower_bound, upper_bound}：上下限都是可选的量词。如果未指定lower_bound，默认值为0。如果未指定upper_bound，默认值为无限大。
    • {fixed_number}：固定量词，取值必须是大于 0 的整数。表示相同的上下限。例如，{2}等于{2,2}。
    • *：等价于{0,}，表示 0 次或更多次的匹配。
    • +：等价于{1,}，表示 1 次或更多次的匹配。

    search_condition¶

    从上一个语句中提取子数据集的布尔值表达式。更多信息，请参见 WHERE 子句。

    element_variable_reference¶

    表示图元素的变量名称。

    path_variable_reference¶

    表示路径的变量名称。
