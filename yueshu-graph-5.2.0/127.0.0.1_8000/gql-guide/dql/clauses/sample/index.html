    SAMPLE¶

    SAMPLE子句用于在遍历过程中对边采样。在处理超大数据集时，可通过该子句选取较小且易于处理的样本进行分析。

    工作原理¶

    取决于遍历的方向，SAMPLE 子句可分为两种类型：向右采样（SAMPLE RIGHT）和向左采样（SAMPLE LEFT）。目前仅支持向右采样。

    SAMPLE 子句必须指定为完整边模式的边模式填充器的组件；且如果边模式填充器中同时指定了 WHERE 子句，则 SAMPLE 子句必须出现在 WHERE 子句之前。关于边模式填充器中各组件出现顺序的更多信息，请参见完整边模式。

    采样对象为包含 SAMPLE 子句的完整边模式所匹配到的边。

    所谓向右采样，也即对锚定点右侧的边进行采样，锚定点即为遍历的起始点。例如，在边模式(v1)<-[e SAMPLE RIGHT]->{1,2}(v2)中，v1即为锚定点。

    采样行为由采样上下文决定，后者由以下四个部分组成：

    • 锚定点
    • 连接锚定点的边的边类型
    • 连接锚定点的边的方向
    • 边模式的每一跳

    在遍历过程中，不同的采样上下文相互独立。针对每个采样上下文，会根据指定的采样方法和采样大小对边进行采样。

    语法¶

    sample_clause
    sampling_method
    sample_size

    sample_clause ::=
    'SAMPLE' 'RIGHT'? sampling_method? sample_size 

sampling_method ::= 
    'FIRST_FETCH' | 'RANDOM' | 'UNIQUE_NEIGHBOR' | 'UNQ_NBR'

sample_size ::=
    unsigned_integer '%'?

    限制条件¶

    对于图和临时图，限制条件如下：

    • SAMPLE 子句不能用于涉及边匹配的近似最邻近查询。更多信息，请参见最近邻查询。
    • 若推断得到无向的边类型，则不支持采样。无向边类型的方向声明如下：~[ edge_pattern_filler ]~。更多信息，请参见 CREATE GRAPH TYPE。

    此外，以下限制条件仅适用于临时图：

    • 仅当临时图的分片方法设置为SRC或DEFAULT时，才支持采样。更多信息，请参见 CREATE TEMPORARY GRAPH。
    • 不支持对指向左的边模式进行采样，例如(v1)<-[e]-(v2)。

    参数¶

    RIGHT¶

    指定采样时遍历方向的可选关键字。当前仅支持向右采样。

    若未指定该关键字，默认为向右采样。

    sampling_method¶

    对于每个采样上下文，选用的采样方法。有效值包括：

    • FIRST_FETCH：采样前 N 条边。此为默认值。
    • RANDOM：随机采样边。
    • UNIQUE_NEIGHBOR或UNQ_NBR：随机采样连接唯一邻点的边。连接同一对点的平行边会去重后再进行采样。

    sample_size¶

    对于每个采样上下文，采样的大小。支持以下两种指定格式：

    • 一个无符号整数，代表最大采样数。此时，该无符号整数必须为正整数。
    • 一个后跟百分号（%）的无符号整数，代表采样百分比。此时，该无符号整数必须在[0, 100]范围内。若计算结果为小数，则向上取整。

    示例¶

    以下示例基于示例数据集。

    采样前 2 条边¶

    '''
USE movie
MATCH (v1{id: 18})<-[e SAMPLE RIGHT FIRST_FETCH 2 ]-(v2) 
RETURN e, labels(v1)
'''

    返回：

    +----------------------------------------------------------------------+------------+
| e                                                                    | labels(v1) |
+----------------------------------------------------------------------+------------+
| (288231381174059009)-[0@Act:Act{}]->(289298959719989249)             | [Movie]    |
| (288231428418699265)-[0@Act:Act{}]->(289298959719989249)             | [Movie]    |
| (288725895118585857)-[0@Direct:Direct{}]->(289298959719989249)       | [Movie]    |
| (289078181221105665)-[0@WithGenre:WithGenre{}]->(289580434696699907) | [Genre]    |
| (289107795020611585)-[0@WithGenre:WithGenre{}]->(289580434696699907) | [Genre]    |
+----------------------------------------------------------------------+------------+

    上述返回显示存在 3 个采样上下文，具体如下：

    1. 对于标签为Movie且 ID 为18的锚定点，采样了两条边类型为Act的边。
    2. 对于标签为Movie且 ID 为18的锚定点，采样了一条边类型为Direct的边。原因是数据集中仅存在一条此类边。
    3. 对于标签为Genre且 ID 为18的锚定点，采样了两条边类型为WithGenre的边。
