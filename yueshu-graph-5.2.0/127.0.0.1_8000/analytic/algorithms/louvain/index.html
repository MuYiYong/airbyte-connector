    Louvain 社区发现¶

    本文介绍 Louvain 算法，并演示如何用 GQL 实现该算法。

    算法介绍¶

    Louvain 算法广泛用于在大型网络中检测社区结构。该算法旨在通过最大化网络的模块度来识别点之间的紧密连接群体，揭示复杂网络中的模块化结构。模块度衡量的是社区内部连接的密集程度与社区之间连接的稀疏程度，通过比较两者，算法能够识别出网络中点间更加紧密相连的群体。通过不断迭代，Louvain 算法能够有效地提升网络的模块度，逐步合并和调整社区划分，从而发现网络中潜在的高质量社区结构。

    在 Louvain 算法中，主要使用两类模块度公式：全局模块度公式（Q）和单一社区模块度贡献公式（Qc）。这两类公式各有其重要作用：

    • Q 用于优化和评估整个网络的社区结构，是衡量整个网络的社区划分质量的标准。Q 值越高，表示社区结构更加显著，即社区内部的连接密度相对于随机连接的预期密度更高。计算公式如下：

      \[ Q = \frac{1}{2m} \sum_{i=1}^N \sum_{j=1}^N \left[ A_{ij} - \frac{k_i k_j}{2m} \right] \delta(c_i, c_j) \]

      其中：

      • \(A_{ij}\)：点i与点𝑗之间的边权（如果两点间有边则为边的权重，没有则为 0；在无权网络中，有边记为 1，无边记为 0）。
      • \(k_i\)、\(k_j\)：点i和点j的总边权（或度数，即与该点相连的所有边的权重之和）。
      • \(m\)：图中所有边的总权重之和的一半。
      • \(c_i\)、\(c_j\)：点i和点j所属的社区编号（即每个点的社区分配）。
      • \(\delta(c_i, c_j)\)：当点i和点j属于同一个社区时取值为 1，否则为 0。

    • Qc 则用于量化单个社区对整体模块度的贡献，其聚焦于某一个特定社区在当前分区中的影响力或质量，有助于进一步分析和细化社区内部结构。计算公式如下：

      \[ Q_c = \frac{\Sigma_{in}}{2m} - \left( \frac{\Sigma_{tot}}{2m} \right)^2 \]

      其中：

      • \(\Sigma_{in}\)：社区c内的边权之和。
      • \(\Sigma_{tot}\)：社区c中所有点的总边权之和。
      • \(m\)：图中所有边的总权重之和的一半。

    更多关于 Louvain 算法的信息，请参见 Louvain 算法。

    数据模型¶

    为了用 GQL 实现 Louvain 算法，将基于以下数据模型来创建图类型：

    • 人员表示为类型为Person的点。每个点分配有一个唯一的id。
    • 人员之间的关系编码为类型为Knows的边。边属性为weight，其体现了关系的强度，用于在 Louvain 算法中计算模块度。

    有关图类型的完整定义，请参见准备图数据

    过程实现¶

    为了检测图中的社区结构，将创建两类过程以集合驱动的松弛方式来实现 Louvain 算法。

    所创建的过程分别为Louvain 基本点移动过程和 Louvain 跨邻居聚合点移动过程。有关这些过程的完整定义，请参见创建过程。下文将介绍过程的关键组成部分及其工作原理。

    所创建的两个过程均接受以下输入参数：

    • max_iter：最大迭代次数。

    此外，这两个过程内部均定义了以下变量：

    变量名 类型 说明
    total_weight 全局 SumAgg 聚合值变量 图中所有边权之和的一半；用于计算模块度。
    node_weight 点绑定 SumAgg 聚合值变量 每个点的边权（度）。
    node_community_weight 点绑定 SumAgg 聚合值变量 每个点与每个社区的总边权（用于评估点移动到不同社区时的模块度增益）。
    community_weight 全局 SumAgg 聚合值变量 每个社区的总边权，由社区中心点维护。
    community_id 点绑定 SumAgg 聚合值变量 每个点当前所属的社区 ID。
    best_move 点绑定 TopKAgg 聚合值变量 每个点在本次迭代中的最佳移动候选（最大delta_q及相应的社区 ID）。
    result_file 文件变量 保存最终点社区划分结果的文件，包括点 ID 及其社区 ID。
    new_active 全局 OrAgg 聚合值变量 全局布尔标记，表示本轮是否发生更新；决定是否继续迭代。
    iter 原始值变量 算法执行后迭代的次数。
    delta 全局 SumAgg 聚合值变量 针对每个点或社区s向邻居t的社区的每个可能移动，将通过delta衡量该移动在模块度上的预期变化。在计算delta时，会综合考虑s与t之间的连接强度（边权）以及社区t的总权重变化。如果delta为正，说明这次移动可以提升模块度，使社区划分更加合理。

    除以上变量外，Louvain 跨邻居聚合点移动过程还定义了以下变量：

    变量名 类型 说明
    community_members 点绑定 ListAgg 聚合值变量 每个社区内成员点 ID 的列表，由社区中心点维护。
    community_center 原始值变量 每个点当前所属社区的 ID。作为社区中心点的引用，用于每次迭代时聚合社区级属性（如总权重和成员列表）。

    以下步骤概述了对应过程的逻辑：

    1. 初始化点和社区信息，包括：

      • 遍历每条边，计算图的总边权（total_weight）以及每个点的边权（node_weight）。
      • 将每个点的初始community_id设置为其自身 ID。
      • 将每个社区的总边权聚合到community_weight中。

    2. 当new_active为true且当前迭代次数小于最大迭代次数（由max_iter指定）时重复执行：

      1. 将new_active设为false，并增加迭代计数器。

      2. 对于每条边(s-t)，根据邻居点t当前的社区 ID，将其边权聚合到点s的node_community_weight中。这样可以为每个点构建一个相邻社区 ID 到其连接边权之和的映射。

      3. 对每条符合条件的边(s-t)，计算点s若移动到邻居社区t时的模块度增益（delta）。该过程利用total_weight、node_weight、node_community_weight和 community_weight等各类权重来估算模块度变化。

      4. 对每个点s，选择模块度增益（best_move）最大的移动。如果该移动的增益（delta_q > 0），则更新点s的社区 ID，调整community_weight中的权重，并将new_active设为true，表示可能还有进一步优化空间。

      5. 清空临时聚合映射（node_community_weight和best_move），为下一次迭代做准备。

    3. 收敛后或达到迭代上限后，将每个点的 ID 及其最终社区 ID 导出到result_file。

    1. 初始化点和社区信息，包括：

      • 遍历每条边，计算图的总边权（total_weight）以及每个点的边权（node_weight）。
      • 对每个点，将其初始community_id设置为自身element_id。

    2. 当new_active为true且当前迭代次数小于最大迭代次数（由max_iter指定）时重复执行：

      1. 将new_active设为false，并增加迭代计数器。

      2. 对每个点，重置其社区的总边权（community_weight），并清空社区成员列表（community_members）。

      3. 对每个点，通过将自身边权加入社区总边权，并将自身加入社区成员列表，更新其社区信息。

      4. 对每个社区，遍历其成员点，并将社区的总边权分配给每个成员（自身除外）。

      5. 对每条边，将起始点到目标点社区的边权聚合到node_community_weight中。该过程用于跟踪每个点与各社区的连接强度。

      6. 对每条符合条件的边，计算起始点移动到目标点社区时的模块度增益（delta_q），并为每个点记录最大模块度增益(best_move)。每次迭代仅考虑单向移动（避免同时交换）。

      7. 对每个点，从其邻居中选择最佳移动（最大正模块度增益）。若存在该移动，则更新点的社区 ID，并将new_active设为true。

      8. 清空临时映射（node_community_weight和best_move），为下一次迭代做准备。

    3. 收敛后或达到迭代上限后，将每个点的 ID 及其最终社区 ID 导出到result_file。

    操作步骤¶

    以下步骤演示如何在悦数图分析中运行 Louvain 算法。

    Note

    也支持在悦数图数据库中运行该算法。此时，只支持创建单机模式的临时图。更多信息，请参见 CREATE TEMPORARY GRAPH。

    1. 准备图数据¶

    1. 创建一个与数据模型匹配的图类型。

      '''
  CREATE GRAPH TYPE IF NOT EXISTS louvain_graph_type {
      NODE TYPE Person (LABEL Person {id INT PRIMARY KEY}),
      EDGE TYPE Knows (Person)~[LABEL Knows {weight DOUBLE, MULTIEDGE KEY ()}]~(Person)
    }
'''

    2. 创建过程¶

    以下将创建两种实现 Louvain 算法的过程：Louvain 基本点移动过程和 Louvain 跨邻居聚合点移动过程。关于过程实现的更多信息，请参见过程实现。

    这两种过程均为 Louvain 算法中的点移动阶段，其目标是通过将点重新分配到不同社区来最大化模块度。在这些过程中，是否移动点由模块度增益的变化量（delta）计算结果决定。

    这两种过程的主要区别在于社区信息的管理方式：

    • Louvain 基本点移动过程将社区权重作为全局变量维护，并在点移动时直接更新这些权重，而不显式追踪社区成员列表。
    • Louvain 跨邻居聚合点移动过程则在每个点本地进行社区权重和成员列表管理，并通过在社区成员之间聚合和传播权重来同步信息。
    '''
CREATE OR REPLACE PROCEDURE louvain_basic(max_iter INT) RETURNS () AS {
        VALUE      total_weight          SumAgg<DOUBLE> = 0
        VALUE      community_weight      MapAgg<INT, SumAgg<DOUBLE>>
        NODE VALUE node_weight           SumAgg<DOUBLE> = 0
        NODE VALUE node_community_weight MapAgg<INT, SumAgg<DOUBLE>>
        NODE VALUE community_id          SumAgg<INT> = 0
        NODE VALUE best_move             TopKAgg<1, delta_q DOUBLE DESC, community_id INT DESC>
        FILE       result_file           {id INT64, community_id INT}= DATAFILE {
                                        PATH: "file:///home/user1/test/louvain_output", 
                                        FORMAT: "CSV"
                                        }
        VALUE      new_active            OrAgg = true
        VALUE      iter = 0

        // 初始化
        MATCH (s)-[e]-(t)
        PER PATH {
          SET @total_weight += e.weight
          SET s.@node_weight += e.weight
        }

        SET @total_weight = @total_weight / 2
        LOG_INFO("total_weight = ", @total_weight)

        MATCH (s)
        PER NODE (s) {
          SET s.@community_id = s.id
          SET @community_weight += TUPLE(s.id, s.@node_weight)
        }

        // 迭代直至无更新
        WHILE iter < max_iter AND @new_active THEN  {
          LOG_INFO("----------- iter == ",iter, " -----------")
          SET @new_active = false
          SET iter = iter + 1

        // 计算每个点对各社区的贡献
        MATCH (s)-[e]-(t)
        PER PATH {
          SET s.@node_community_weight += TUPLE(t.@community_id, e.weight)
        }

        // 选择最佳移动方向
        MATCH (s)-[e]-(t)
        WHERE s.id < t.id AND iter % 2 = 0 OR s.id > t.id AND iter % 2 = 1
        PER PATH {
          VALUE delta = s.@node_community_weight.get(t.@community_id) - s.@node_community_weight.get(s.@community_id)
            + 1 / @total_weight * s.@node_weight * (@community_weight.get(s.@community_id) - @community_weight.get(t.@community_id))

          SET s.@best_move += RECORD {
            delta_q: delta,
            community_id: t.@community_id
          }
        }

        // 移动点至新社区
        MATCH (s)
        PER NODE (s) {
          VALUE move_list = s.@best_move
          IF length(move_list) <> 0 THEN {
            VALUE move = move_list[0]
            IF move.delta_q > 0 THEN {
              SET @new_active += true
              SET @community_weight += TUPLE(s.@community_id,-s.@node_weight)
              SET s.@community_id = move.community_id
              SET @community_weight += TUPLE(s.@community_id,s.@node_weight)
            }
         }
          SET s.@node_community_weight.clear()
          SET s.@best_move.clear()
        }
      }

        // 收集结果
        MATCH (s)
        PER NODE (s) {
          EXPORT s.id, s.@community_id INTO result_file
        }
    }
'''

    '''
CREATE OR REPLACE PROCEDURE louvain_cross_neighbor(max_iter INT) RETURNS () AS {
        VALUE      total_weight          SumAgg<DOUBLE> = 0
        NODE VALUE node_weight           SumAgg<DOUBLE> = 0
        NODE VALUE node_community_weight MapAgg<INT, SumAgg<DOUBLE>>
        NODE VALUE community_weight      SumAgg<DOUBLE> = 0
        NODE VALUE community_id          SumAgg<INT> = 0
        NODE VALUE community_members     ListAgg<INT>
        NODE VALUE best_move             TopKAgg<1, delta_q DOUBLE DESC, community_id INT DESC>
        FILE       result_file           {id INT64, community_id INT}= DATAFILE {
                                         PATH: "file:///home/user1/test/louvain_output2", 
                                         FORMAT: "CSV"
                                         }
        VALUE      new_active            OrAgg = true
        VALUE      iter = 0

        // 初始化
        MATCH (s)-[e]-(t)
        PER PATH {
          SET @total_weight += e.weight
          SET s.@node_weight += e.weight
        }

        SET @total_weight = @total_weight / 2
        LOG_INFO("total_weight = ", @total_weight)

        MATCH (s)
        PER NODE (s) {
          SET s.@community_id = element_id(s)
        }

        // 迭代直至无更新
        WHILE iter < max_iter AND @new_active THEN  {
          LOG_INFO("----------- iter == ",iter, " -----------")
          SET @new_active = false
          SET iter = iter + 1

        MATCH (s)
        PER NODE (s) {
          SET s.@community_weight = 0
          SET s.@community_members.clear()
        }
        PER NODE (s) {
          VALUE community_center = s.@community_id
          SET NODE(community_center).@community_weight += s.@node_weight
          SET NODE(community_center).@community_members += element_id(s)
        }

        MATCH (s) WHERE s.@community_members.size() > 0
        PER NODE (s) {
          VALUE i = 0
          VALUE members = s.@community_members
          VALUE len = length(members)
          WHILE i < len THEN {
            VALUE member = members[i]
            SET i = i + 1
            IF member = element_id(s) THEN {
                CONTINUE
            }
            SET NODE(member).@community_weight += s.@community_weight
            }
        }

        // 计算每个点对各社区的贡献
        MATCH (s)-[e]-(t)
        PER PATH {
          SET s.@node_community_weight += TUPLE(t.@community_id, e.weight)
          // LOG_INFO(s.id," weight += [",t.id, ",", t.@community_id, ",", e.weight,"]")
        }

        // 选择最佳移动方向
        MATCH (s)-[e]-(t)
        WHERE s.id < t.id AND iter % 2 = 0 OR s.id > t.id AND iter % 2 = 1
        PER PATH {
          VALUE delta = s.@node_community_weight.get(t.@community_id)
                      - s.@node_community_weight.get(s.@community_id)
                      + 1 / @total_weight * s.@node_weight * (s.@community_weight - t.@community_weight)
          SET s.@best_move += RECORD {
            delta_q: delta,
            community_id: t.@community_id
            }
        }

        // 移动点至新社区
        MATCH (s)
        PER NODE (s) {
          VALUE move_list = s.@best_move
          IF length(move_list) <> 0 THEN {
            VALUE move = move_list[0]
            IF move.delta_q > 0 THEN {
              // LOG_INFO(s.id," move to ",move.community_id, " delta: ",move.delta_q)
              SET @new_active += true
              SET s.@community_id = move.community_id
              }
            }
              SET s.@node_community_weight.clear()
              SET s.@best_move.clear()
            }
        }

        // 收集结果
        MATCH (s)
        PER NODE (s) {
          EXPORT s.id, s.@community_id INTO result_file
        }
    }
'''
